**输入需求**：`iterations/0.0.2/product/prd-chat-sessions-sqlite.md`、`user-stories-chat-sessions-sqlite.md`  
**基线布局与按钮**：`docs/design/spec-chat-layout-buttons.md`（0.0.1）、`src/components/chat/ChatApp.tsx`  
**基线对话行为**：`docs/design/spec-ai-agent.md`

## 1. 设计决策（D1～D4 定稿）

### D1 信息架构：会话列表 + 主对话区

| 决策 | 内容 |
|------|------|
| **桌面（≥ sm，建议 `640px` 及以上）** | **左侧固定会话栏** + **右侧主区**（顶栏 / 消息列表 / 底栏输入区）。侧栏宽度约 **`14rem`～`16rem`**（`w-56`～`w-64`），与主区间 **`border-r`** 分隔，背景与主区一致体系（`zinc-50` / `zinc-950`）。 |
| **窄屏（&lt; sm）** | 默认**隐藏侧栏**；顶栏左侧增加 **「会话」** 文字按钮或图标按钮，点击打开 **抽屉（从左侧滑入）** 覆盖主区之上，带半透明遮罩；选中会话或点击遮罩关闭抽屉。 |
| **侧栏内容** | 顶部：**「新对话」** 主按钮（与 violet 主按钮风格一致，可全宽）。其下：可滚动 **会话列表**（按**更新时间倒序**，最新在上）。 |
| **列表项** | 单行标题（**截断**，`truncate`）+ 可选次要信息（相对时间 `更新于…`，由前端格式化）；**当前会话**项使用 **左边框高亮**（`border-l-2 border-violet-500`）或 **背景略深**（`bg-zinc-200/ dark:bg-zinc-800`），与 PRD「当前会话明确指示」一致。 |

### D2 状态：空态、加载、错误

| 状态 | 表现 |
|------|------|
| **应用首次加载（拉列表）** | 侧栏列表区显示 **骨架或简短「加载中…」**；主区可显示全局轻量加载，避免双空白。 |
| **列表加载失败** | 侧栏内 **错误文案 + 重试**；主区仍可保留上次成功会话或空态说明（实现可选，优先侧栏可恢复）。 |
| **无任何会话** | 侧栏仅「新对话」；主区走 **空态**（见下「与单会话空态关系」）。 |
| **有会话但未选中**（边界） | 产品期望：进入页时 **自动选中最近更新会话** 或 **自动创建一条**；设计推荐 **默认选中列表首条**（最新），若无则等同「新会话」流程。 |
| **选中会话但消息为空** | 与现网 **「开始对话」** 空态一致（图标 + 说明文案）；文案可微调为「向当前会话发送第一条消息」。 |
| **切换会话** | 主区消息列表 **先清空或显式加载占位**，再渲染目标会话消息，**避免**旧消息与新区短暂叠显（防串线，对应 US-2 AC）。 |
| **发送中 / busy** | 保持现网：输入禁用、顶栏模型与清空等禁用策略不变；**不允许**在 busy 时切换会话（禁用侧栏项与抽屉内项，或整侧栏 `pointer-events-none`），简化竞态（若产品坚持可切换，需后端文档定义未完成流的归属，**本迭代推荐禁止切换**）。 |

### D3 交互：新建与切换

| 决策 | 内容 |
|------|------|
| **新建对话** | 点击「新对话」→ 调用 **创建会话**（或前端临时 ID + 首条消息时创建，以后端定稿为准）→ **切换当前会话**到新会话 → **焦点落入输入框**（`textarea.focus()`）。 |
| **切换历史** | **无需**「未保存」确认（无草稿模型）。 |
| **清空对话** | 语义调整为 **清空当前会话的消息**（与 PRD 一致：持久化后「清空」应对 DB 中当前 session 生效）；侧栏该条会话仍可保留（空消息）或仍显示标题，由实现决定；**设计建议**：清空后会话仍在列表中，标题不变或显示「空会话」。 |

### D4 视觉与现有 Chat 壳一致

- 顶栏、消息气泡、footer 输入区、图标按钮（发送 / 清空 / 重试）**延续** `ChatApp.tsx` 与 0.0.1 spec：**线框图标 + 中文**、`violet` 强调色、`rounded-xl` / `rounded-2xl` 层级。
- 侧栏内「新对话」与列表项 hover/active 使用 **同一 zinc 阶梯**，不引入新品牌色。

## 2. 用户故事映射

| 故事 | 设计落点 |
|------|----------|
| US-1 | 侧栏「新对话」+ 新项出现在列表顶部（或创建后排序规则与后端一致） |
| US-2 | 列表点击 + 当前项高亮 + 切换时主区加载策略 |
| US-3 | 无单独 UI；依赖刷新后会话列表与消息从服务端拉取 |
| US-4 / US-5 | 不在 UI 层体现协议细节；前端发送时 **仅** 带 `sessionId`、本轮正文、`provider`/`model`（设计约束与开发一致） |
| US-6 | 无 UI；文档见后端 |

## 3. 前端状态与数据流（供实现对照）

以下为**建议**，具体字段名以实现为准：

- **`activeSessionId`**：当前会话；切换会话时更新。
- **`sessions[]`**：`{ id, title, updatedAt }` 或等价；标题规则：**首条用户消息前 30～40 字**，无消息时 **「新对话」** 或 **「未命名会话」**。
- **消息列表**：仅展示 **当前会话** 的消息；切换时 **重新请求** 该会话消息或使用缓存策略（若缓存，须与 `activeSessionId` 校验）。

**首屏**：`GET` 会话列表 +（可选）`GET` 当前会话消息，或合并接口——由后端 spec 定稿。

---

## 4. 与后端交接（设计侧约束）

- 对话 **POST** 请求体目标态：**不包含**完整历史数组；含 **`sessionId`**、**本轮用户文本**、`provider`、智谱时 `model`。
- **错误**：`sessionId` 无效时，前端在消息区或 toast 展示服务端错误文案；**不重试**自动换会话。
- **重试**：仍针对**当前轮次** assistant 错误；重试请求仍只带 **sessionId**（及后端要求的 replay 标识若需要），不带全量 history。

---

## 5. 可访问性

- 侧栏列表：每条为 **`button` 或带 `role="tab"` 的自定义控件**（二选一）；当前项 `aria-current="true"` 或 `aria-selected`。
- 「会话」打开抽屉：`aria-expanded`、焦点陷阱（可选）、关闭后焦点回到触发按钮。
- 「新对话」：`aria-label` 与可见文案一致。
