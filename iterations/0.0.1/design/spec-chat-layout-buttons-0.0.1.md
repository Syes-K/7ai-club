# 设计说明：对话页模型位置与操作按钮（version=0.0.1）

**输入需求**：`iterations/0.0.1/product/prd-chat-layout-buttons.md`、`user-stories-chat-layout-buttons.md`  
**基线设计**：`docs/design/spec-ai-agent.md`（未冲突条目仍适用；本节为 **布局与按钮增量定稿**）

---

## 1. 设计决策摘要（定稿 D1～D4）

| 待设计项 | 决策 |
|----------|------|
| **D1 模型位置** | 模型选择 **迁至顶栏右侧工具区**，与 **「清空对话」** 同处一行（或窄屏时自动换行到第二行）。**不再**在 footer 内、输入框上方单独占一行。 |
| **D2 输入区** | Footer **仅保留**：多行 `textarea` + **发送**按钮（右侧、`self-end` 与输入框底对齐）。无额外「模型」行。 |
| **D3 图标** | **线框图标**（`stroke="currentColor"`、`strokeWidth={2}`），与 `BotIcon` / `UserIcon` 一致。发送/清空/重试均采用 **图标 + 可见中文文案**（满足 AC-L2.4，文案即名称）。图标尺寸约 **16×16px**（`h-4 w-4`）。 |
| **D4 状态** | `busy` 时：`select` 禁用、清空禁用、发送禁用（空输入亦禁用发送）；禁用态 **整体 `opacity-40`（或项目既有 disabled 样式）**，图标与文字同步变灰。无消息时清空禁用，逻辑与现网一致。 |

## 2. 页面结构（修订 §2.1 对应区域）

### 2.1 顶栏（`header`）

```
┌─────────────────────────────────────────────────────────────┐
│ [Bot] AI 对话          [模型 ▾ 下拉........]  [🗑 清空对话]   │
└─────────────────────────────────────────────────────────────┘
```

- **左侧**：现有 BotIcon + 标题「AI 对话」，不变。
- **右侧工具组**（`flex flex-wrap items-center justify-end gap-2`）：
  1. **模型**：保留 `<label htmlFor="model-select">` 文案「模型」（`text-xs text-zinc-500`）+ 原生 `<select id="model-select">`（分组 option 不变）。
  2. **清空对话**：按钮内含 **清空图标** + 文案「清空对话」；`disabled={busy \|\| messages.length===0}`。

**对应**：US-L1（AC-L1.1～L1.4）、US-L2（AC-L2.2）

### 2.2 底部输入区（`footer`）

```
┌─────────────────────────────────────────────────────────────┐
│  ┌──────────────────────────────────────┐  ┌─────────────┐  │
│  │ 多行 textarea                         │  │ [➤ 发送]    │  │
│  │                                       │  └─────────────┘  │
│  └──────────────────────────────────────┘                    │
└─────────────────────────────────────────────────────────────┘
```

- **一行主布局**：`flex gap-2`，`textarea` `flex-1`，发送按钮在右侧纵向贴底（`self-end`）。
- **发送按钮**：**纸飞机**线框图标 + 「发送」；`disabled={busy \|\| !input.trim()}`。

**对应**：US-L2（AC-L2.1）、US-2（多行、快捷键，基线 spec §3.3）

### 2.3 空态文案（建议）

- 将引导句由「选择模型后输入问题」改为 **「在顶栏选择模型后输入问题」**（或等价表述），与模型入口位置一致。

### 2.4 消息区内错误气泡 · 重试

- **重试**按钮：**旋转箭头**（refresh）线框图标 + 「重试」；`disabled={busy}`。
- 错误条左侧可保留 ⚠ 或改为小号 **alert 线框图标**（二选一，以前端实现一致为准）。

**对应**：US-L2（AC-L2.3）、US-6

## 3. 图标规范（实现约束）

| 操作 | 隐喻 | 备注 |
|------|------|------|
| 发送 | 纸飞机（向上/向右飞出） | 与「发出消息」心智一致 |
| 清空对话 | 垃圾桶 | 与「删除会话内容」一致，非「单条删除」 |
| 重试 | 环形箭头 refresh | 与「重新请求」一致 |

- **风格**：24×24 `viewBox`，路径 `stroke`，`fill="none"`，`strokeLinecap="round"`，`strokeLinejoin="round"`。
- **颜色**：继承按钮 `text-*` / `currentColor`；主按钮（发送）上为白字则图标同为白色。

---

## 4. 响应式与断点

- **宽屏**：顶栏右侧模型 + 清空同一行；模型 `select` 设 `max-w-[min(100%,14rem)]` 或 `sm:max-w-*`，避免挤压标题（可用 `min-w-0` 于 flex 子项）。
- **窄屏（&lt; ~640px）**：顶栏 `flex-wrap`，标题行满宽后，**第二行**为「模型下拉 + 清空」整行或右对齐，保证 **仍可点选、不被裁切**（AC-L1.3）。
- Footer：textarea 与发送保持横排；极窄时若发送被压缩，发送按钮可 `shrink-0`。

---

## 5. 可访问性

- `select`：**保留** `id` + `label` 关联。
- 所有带图标按钮：**保留可见中文**，无需额外 `aria-label`；若后续改为仅图标，必须补 `aria-label`。
- **焦点**：`focus-visible:ring-2` 或与现网 `ring-violet-500` 一致，顶栏按钮与 footer 发送均需可见焦点环（AC-L2.5）。
