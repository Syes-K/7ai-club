# PRD：后台管理 ProLayout（mix）与侧栏导航（version=0.0.8）

## 1. 需求摘要

### 1.1 背景

`/console` 管理区已具备「应用配置」（`/console`）与「日志」（`/console/logs`）等能力，当前外壳为自定义顶栏 + 横向文字导航（`ConsoleChrome`）。为与 antd-pro 生态一致、便于后续扩展更多管理菜单，需要将**整站管理壳**升级为 **`@ant-design/pro-components` 的 `ProLayout`**，并采用 **`layout="mix"`**（顶栏 + 侧边菜单的混合布局）。

### 1.2 目标

- 管理区呈现统一品牌：**布局标题为「后台管理」**。
- **左侧菜单**展示 **「应用配置」**、**「日志」** 两项，**每项带图标**；点击与现有路由一致（`/console`、`/console/logs`）。
- 保持现有各子页业务功能与数据行为不变；本迭代聚焦**导航与布局壳**的替换与体验对齐。

### 1.3 成功指标

- 用户在 `/console` 与 `/console/logs` 间切换时，依赖侧栏即可完成，且当前页在菜单中有明确选中态。
- 新用户进入管理区能识别整体为「后台管理」，并能从图标+文案理解两项入口含义。

---

## 2. 目标用户与核心场景

| 角色 | 说明 |
|------|------|
| 开发者 / 运维 | 调整应用配置、查看结构化日志排障。 |

**核心场景：**

1. 从对话页进入管理区，看到 **「后台管理」** 与 **mix 布局**，通过**左侧菜单**进入配置或日志。
2. 在配置页与日志页之间来回切换，**选中态与内容区**正确对应。

---

## 3. 功能范围

### 3.1 做（In Scope）

- 在 `/console` 布局层级接入 **`ProLayout`**，`layout` 为 **`mix`**。
- 配置 **`menuDataRender`（或等价方式）** 定义两项：**应用配置**、**日志**，路径与现有一致；**每项配置图标**（具体图标由设计选定或前端在实现说明中约定）。
- 配置 **`title`**（或 ProLayout 约定的主标题字段）为 **「后台管理」**。
- 保留合理的**返回对话**或等价出口（位置与样式由设计在壳层统一规定；若沿用现有「返回对话」链接，行为不变）。
- **响应式**：在窄屏下侧栏可折叠/抽屉等行为遵循 ProLayout 默认或可配置项，由设计注明期望。

### 3.2 不做（Out of Scope）

- 不新增管理菜单项（除上述两项外不扩展路由）。
- 不改变应用配置表单、日志 ProTable/筛选等业务组件逻辑（除非因布局占位导致的必要样式微调，需在实现说明中记录）。
- 不引入新的服务端 API 或鉴权模型（沿用现有 Console 访问假设）。
- 不要求与主站（`/` 对话区）共用同一套 Layout；仅要求 `/console` 下一致。

### 3.3 依赖与假设

- 项目已依赖 **`@ant-design/pro-components`**（内含 `ProLayout`）。
- 继续使用现有 **`ConsoleAntdProvider`**（或等价 ConfigProvider）以保证样式与主题一致，具体整合方式由设计/前端定稿。

## 4. 用户故事与验收标准

### US-1：侧栏进入应用配置

**作为** 管理区用户  
**我想要** 在左侧菜单点击「应用配置」  
**以便** 打开现有配置页

**验收标准：**

- [ ] 菜单项文案为「应用配置」，**带图标**，导航至 `/console`。
- [ ] 当路径为 `/console`（或其后无额外 segment 的约定）时，该项为**选中态**。

### US-2：侧栏进入日志

**作为** 管理区用户  
**我想要** 在左侧菜单点击「日志」  
**以便** 打开现有日志页

**验收标准：**

- [ ] 菜单项文案为「日志」，**带图标**，导航至 `/console/logs`。
- [ ] 当路径为 `/console/logs` 时，该项为**选中态**。

### US-3：布局标题与 mix 形态

**作为** 管理区用户  
**我想要** 看到明确的后台标题与 mix 布局  
**以便** 识别当前处于管理后台而非对话页

**验收标准：**

- [ ] 使用 **`ProLayout`**，**`layout="mix"`**。
- [ ] 主标题展示为 **「后台管理」**（与菜单、内容区同时存在时层级清晰，具体区域由设计定义）。

### US-4：离开管理区

**作为** 管理区用户  
**我想要** 能返回对话主界面  
**以便** 继续聊天

**验收标准：**

- [ ] 提供「返回对话」（或同等语义）入口，跳转至 `/`（或项目当前对话首页路径），行为与改造前一致或更明确。

## 5. 待设计项（交给设计阶段）

1. **信息架构**：mix 顶栏与侧栏分工（Logo/标题区、用户区、折叠控制等）线框。
2. **图标**：两项菜单的图标选型（与 antd icons 一致）。
3. **内容区标题**：是否在 ProLayout `pageTitle` / 子页内保留「应用配置」「日志」等二级标题，避免与「后台管理」混淆。
4. **暗色模式**：与现有 `ConsoleAntdProvider`、全站暗色策略对齐的 ProLayout 区域样式。
5. **窄屏与可访问性**：侧栏折叠、焦点顺序、`aria` 标签等要点。
