# PRD：会话删除与发送按钮布局（version=0.0.3）

## 1. 需求摘要

### 1.1 背景

在 **0.0.2** 已支持多会话、SQLite 持久化与侧栏列表的基础上，用户需要**管理会话生命周期**（删除不再需要的对话），并优化输入区**发送按钮**位置：固定在**输入框区域右下角悬浮**，与主输入路径对齐、减少长输入时的视线移动。

### 1.2 目标用户

使用 AI 对话页的终端用户。

### 1.3 核心场景

1. 用户在会话列表中可对**某一会话执行删除**，删除后该会话及其消息从持久化存储中移除，列表与主区状态正确更新。
2. 用户输入时，**发送**按钮位于**多行输入框容器**的**右下角内侧（悬浮）**，不独占整行 footer 高度，视觉与操作符合「输入为主、发送为辅」的层级。

### 1.4 成功指标

- 删除后刷新或重开页面，被删会话**不再出现**。
- 删除当前正在查看的会话时，界面**不崩溃**，自动切换到合理目标会话（见范围）。
- 发送按钮在常见视口与多行输入高度下**始终可见且可点**（含禁用态样式）。

---

## 2. 功能范围

### 2.1 做（In Scope）

- **删除会话（单条）**：在侧栏（及窄屏抽屉内）每条会话提供**删除入口**；删除前**二次确认**（浏览器 `confirm` 或轻量 Modal，由设计定稿）。
- **数据一致性**：服务端删除 `chat_sessions` 记录并级联删除消息（与现有外键一致）；若当前激活会话被删，**自动选中**列表中**下一条**（按更新时间排序后的逻辑「下一条」）或若无则**新建空会话**（与 0.0.2 首屏策略一致）。
- **发送按钮布局**：输入区为**相对定位容器**；发送按钮**绝对定位**于容器**右下角**（`bottom`/`right` 与内边距由设计标注），**悬浮**于输入框之上或紧贴右下角内侧，**不**再使用「整行 footer 左输入 + 右侧大按钮并列占满」的独占布局（可与多行 `textarea` 重叠区域留白，避免遮挡文字——由设计规定最小 `padding-bottom` 或 `textarea` 底部内边距）。

### 2.2 不做（Out of Scope）

- **批量删除**、回收站/软删除。
- **拖拽排序**会话列表。
- **修改** SSE、模型选择、0.0.2 已定义的 session 发送契约（除新增删除会话 API 外）。

### 2.3 依赖与假设

- 沿用 `ChatStore` 抽象；SQLite 实现增加 `deleteSession(id)`（或等价）并由 REST 暴露。
- 前端仍基于 `ChatApp` / `SessionSidebar` 结构扩展。
