# 用户故事：会话删除与发送按钮布局（version=0.0.3）

## US-1：删除指定会话

**作为** 对话用户，**我想要** 在会话列表里删除某条会话，**以便** 清理不需要的历史对话。

### 验收标准（AC）

- [ ] 每条会话项提供**删除**入口（与列表项可区分，不误触打开会话）。
- [ ] 删除前出现**二次确认**；取消则列表与当前对话不变。
- [ ] 确认后该会话从列表消失；数据库中该会话及消息不可再被 `GET` 到。
- [ ] **删除进行中**该条或全局有明确 loading/禁用，避免重复提交。

---

## US-2：删除当前正在查看的会话

**作为** 对话用户，**我想要** 在删除「当前会话」后仍停留在可用状态，**以便** 继续聊天或新建会话。

### 验收标准（AC）

- [ ] 若删除的是**当前激活**会话，删除成功后**自动切换**到另一会话（优先列表中**剩余**的最近更新项）或**自动创建并选中**新会话（当列表为空时）。
- [ ] 主消息区与 `sessionId` 一致，不出现指向已删 id 的请求。

---

## US-3：发送按钮位于输入框右下角悬浮

**作为** 对话用户，**我想要** 发送按钮固定在输入框区域的**右下角**，**以便** 长文本输入时仍能快速发送。

### 验收标准（AC）

- [ ] 发送按钮相对**输入区容器**定位于**右下角内侧**（悬浮/叠层），而非与 `textarea` 左右并排占满 footer 一整行。
- [ ] 调整 `textarea` 行高或窗口大小时，发送按钮仍保持在输入区**右下角**（同一容器内）。
- [ ] `textarea` 为可编辑区保留足够底部内边距或占位，**避免**正文被按钮大面积遮挡（具体像素由设计标注，实现可验收「末行文字可读」）。
- [ ] 禁用态（`busy`、无输入等）视觉与 0.0.1 规范一致，仅位置变化。
