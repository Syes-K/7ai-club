# AI Agent 对话功能 — 设计说明（spec-ai-agent）

本文档承接 `docs/product/prd-ai-agent.md` 与 `docs/product/user-stories-ai-agent.md`，供服务端与前端实现对照。需求阶段已确认，本文为阶段 2 产出。

---

## 1. 流程与页面

### 1.1 主流程

1. 用户进入**唯一主页面**（对话页），默认选中模型：**智谱 `glm-4-flash`**（与 `src/lib/chat/route-key.ts` 一致）。
2. 用户可在**模型下拉**中切换 **DeepSeek** 或 **智谱下某一 GLM**；切换仅影响**之后**发送的消息。
3. 用户在**多行输入框**输入问题，通过**发送按钮**或 **Ctrl + Enter**（macOS：**⌘ + Enter**）发送；**Enter** 换行。
4. 发送后：用户消息立即出现在列表底部；同位置出现 AI 消息占位 + **加载态**；首段流式到达后加载态消失，内容流式追加。
5. 用户可连续多轮发送；刷新或离开页面后历史清空（首版不持久化）。
6. 出错时：在**该轮 AI 位置**展示错误条，提供**重试**；页级提供**清空对话**。

**页面跳转**：首版无子路由，单页完成全部交互。

| 对应故事 | US-1～US-6 |

## 2. 页面结构（对话页）

### 2.1 布局（桌面优先，响应式）

- **顶栏**：左侧产品/对话标题 + 小图标；右侧可选「清空对话」文字按钮 + 当前模型摘要（或与下拉里选中一致，二选一避免重复）。
- **主体**：可滚动**消息列表**（`flex-1 min-h-0 overflow-y-auto`），底部**固定输入区**（输入框 + 发送 + **模型选择下拉**）。
- **断点**：窄屏时顶栏与输入区纵向不挤压可读性；消息气泡最大宽度约 `min(100%, 42rem)`，用户消息右对齐、AI 左对齐（或上下区分样式）。

### 2.2 关键页面/视图列表

| 视图 | 说明 | 对应 US |
|------|------|---------|
| 空态 | 无消息时：插图/图标 + 简短引导文案 + 可选示例提示 | US-2 |
| 仅有用户消息 | 列表中最后一条为用户消息，其下为 AI 加载或流式气泡 | US-2, US-3, US-5 |
| 多轮对话 | 用户/AI 交替按时间顺序展示 | US-4 |
| 错误态 | 该轮 AI 区域显示错误卡片/条 + 重试 | US-6 |

### 2.3 迭代 0.0.1（模型位置与操作按钮）

自 **version=0.0.1** 起，对话页 **布局与按钮** 以 **`docs/design/spec-chat-layout-buttons-0.0.1.md`** 为定稿增量：**模型选择迁至顶栏右侧**（与「清空对话」同组），**底部输入区仅保留 textarea + 发送**；发送/清空/重试带线框图标 + 中文文案。上文 §2.1 中「底部固定输入区…**模型选择下拉**」在 0.0.1 中由该文档替代；模型下拉交互细节仍见 §3.1。

## 3. 组件与交互

### 3.1 模型选择（下拉分组）

- **控件**：`<select>` 或自定义下拉，内含 **`<optgroup label="DeepSeek">`** 与 **`<optgroup label="智谱 GLM">`**。
- **选项文案**：智谱各 `modelId` 对应简短标题 + 括号内特点（速度/旗舰/多模态等），数据源自 `src/lib/chat/zhipu-models.ts`（与 PRD §4 一致）。
- **选中态**：下拉显示当前 `provider + model`；切换后**不清空**历史消息（US-1 AC1.3）。
- **对应**：US-1（AC1.1～AC1.4）

### 3.2 消息列表

- **用户消息**：右侧或区分背景/边框；可选小头像图标。
- **AI 消息**：左侧；流式过程中同一条气泡内文字递增；结束后完整展示。
- **自动滚动**：新内容或流式增量时，若用户未手动上滚查看历史，则**滚动到底部**（US-3 AC3.3）；实现上可在列表容器 `scrollTop = scrollHeight`（可加「是否在底部」判断以减少打扰）。
- **对应**：US-3、US-4、US-5、US-6

### 3.3 输入框与发送

- **多行** `textarea`，占位符说明快捷键。
- **发送**：按钮 + **Ctrl/⌘+Enter**；请求进行中**禁用**发送与输入或仅禁用发送（与 PRD AC2.4 一致：避免重复请求）。
- **发送后**：清空输入框（US-2 AC2.3）。
- **对应**：US-2

### 3.4 加载状态

- AI 占位气泡内：**骨架/转圈 +「AI 正在思考…」** 等文案（US-5）；首 token 到达即移除加载-only 层，保留同一条气泡接流式正文。
- **对应**：US-5

### 3.5 错误状态

- 展示**可读错误信息**（网络、HTTP、业务文案），不依赖控制台（US-6 AC6.1）。
- 错误挂在**该轮** AI 消息位置（US-6 AC6.2）。
- **重试**：复用同一条用户消息与当前上下文再次请求（US-6 AC6.3）。
- **清空对话**：顶栏或输入区附近，清空后新会话（US-6 AC6.4）。

## 4. 视觉与可访问性

- **对比度**：正文与背景满足基本可读；错误用语义色（如红/琥珀）但不仅依赖颜色（可配图标）。
- **焦点**：输入框、按钮、下拉可键盘聚焦；`textarea` 与 `label` 关联（若用自定义控件需 `aria-*`）。
- **动效**：流式文字出现可为即时无动效或轻微过渡，避免眩晕；加载用轻微 pulse 即可。
