# 设计说明：管理端日志查看与检索（version=0.0.6）

本文档承接 `iterations/0.0.6/product/prd-admin-log-viewer.md` 与 `user-stories-admin-log-viewer.md`，供 **backend** / **frontend** 实现与验收对照。

---

## 1. 设计目标与一致性

- **场景**：开发/排障——快速按维度缩小范围、按 `requestId` 串联、单条结构化阅读。
- **视觉与布局**：与现有 Console（`/console`）一致——`zinc` 背景与边框、顶栏标题 + 右侧「返回对话」链接、主内容区 `max-w` 与内边距节奏可参考配置页；**琥珀色安全提示条**延续（内容针对日志敏感性与部署访问边界；**本迭代无**日志专用 API 密钥 UI）。
- **隐喻**：产品是「受控的日志检索与详情」，不是通用文件浏览器；文案避免强调服务器路径。

---

## 2. 信息架构与页面流程

### 2.1 入口与路由

| 决策 | 说明 |
|------|------|
| **主入口** | 独立路由 **`/console/logs`**（与 `/console` 配置页并列，同属 Console 域）。 |
| **子导航** | 在 **`/console` 与 `/console/logs` 顶栏下方**增加一行轻量 **Console 子导航**（Tab 或文字链接）：「应用配置」→ `/console`，「日志」→ `/console/logs`。当前页高亮；移动端可收折为下拉或横向滚动，保证可点达即可（对应 PRD D-07）。 |

**追溯**：D-01（入口与布局）、LOG-01。

### 2.2 页面清单

1. **日志列表页**（`/console/logs`）：筛选区 + 结果表格 + 分页/加载更多；选行打开详情。
2. **单条详情**：不单独占路由（避免刷新丢列表上下文）；见下文「详情载体」。

### 2.3 详情载体：**右侧抽屉（Drawer）**

| 方案 | 取舍 |
|------|------|
| **抽屉（推荐）** | 列表仍可见，便于对照多行、`requestId` 复制后回到列表；宽屏信息密度高；关闭后易恢复列表与筛选状态（LOG-04）。 |
| 全屏 Modal | 遮挡列表，对照成本高。 |
| 行内展开 | 长 JSON 与多区块会撑乱表格行高，键盘与屏幕阅读器遍历变差。 |

**行为摘要**：

- 从表格行点击「查看」或行主区域（除交互控件外）打开抽屉；URL **可选**同步 `?detail=<id>`（实现选做）；若不同步，关闭抽屉须保留筛选条件（LOG-04）。
- 抽屉宽度：桌面约 **min(560px, 40vw)**，窄屏 **全屏或近全屏**（D-07）。

**追溯**：D-02、D-04、LOG-04。

## 3. 筛选区交互（D-03 / LOG-03）

### 3.1 布局

- 筛选区位于表格上方卡片内（白底/浅底、圆角、与配置表单区域风格一致）。
- 控件分组建议：**时间一行**（起 + 止 + 说明文案）→ **维度一行**（level、event、requestId、keyword）→ **操作一行**（应用、重置）。

### 3.2 各控件定义

| 控件 | 类型与行为 | 说明 |
|------|------------|------|
| **绝对时间区间** | 两个 **日期时间** 输入：**开始**、**结束**，展示精度到 **分钟**（秒固定为 `00` 或由实现统一归零，须在 API 文档写明）。 | 标签旁短文案：**「筛选依据为每条日志的 `ts`（UTC 时刻）；此处为浏览器本地时区展示。」** 见第 4 节时区约定。 |
| **level** | **多选**：复选框组或多选下拉；选项来自服务端发现的取值 + 固定 **「未标注」**（见 3.4）。 | 与 `event`、时间、`requestId`、关键词组合为 **交集**。 |
| **event** | **组合框（Combobox）**：支持 **从已发现 `event` 列表多选** + **允许自由输入自定义 event 字符串**（回车或「添加」芯片）。已选 event 以 **可删除标签（chip）** 展示。 | **同一维度内：多条 event 为 OR**；与其他维度为 **AND**。见 3.3。 |
| **requestId** | 单行文本；占位符说明匹配规则（**精确**或**前缀**，由实现二选一并在此写死一种且在帮助文案中明示——与故事 LOG-02 一致）。 | 空 = 不限。 |
| **关键词** | 单行搜索；辅助文案说明范围（如：**整行 JSON 序列化文本**或实现选定的字段子集，须固定并文档化）。 | 空 = 不限。 |

### 3.3 组合语义（须在界面可见）

- 在筛选区底部或「应用」按钮旁增加 **一行辅助说明**（小号字、muted）：
  - **「多个 event 之间为“或”；与 level、时间范围、requestId、关键词之间为“且”。」**
- 与 PRD / 故事一致：**维度间交集**；**event 多选为 OR**（本设计已定稿）。

**追溯**：LOG-03、D-03。

### 3.4 level 缺失（字段不存在或 null）

- **表格**：显示 **「—」**（em dash），不显示空白。
- **筛选**：
  - 多选中包含 **「未标注」** 选项：匹配无 `level` 或不可解析的行。
  - **「全部」/不选任何 level**：语义为 **不限 level**（包含未标注与所有已知 level）。
  - 若实现采用「必须至少选一个」则禁止——应保持可不选即不限。

**追溯**：PRD 4.1、LOG-03。

### 3.5 查询触发：**应用 + 重置**（非自动搜索）

| 决策 | 说明 |
|------|------|
| **应用** | 点击后提交当前表单为一次查询；表格进入加载态并刷新结果。避免输入过程中频繁请求（大日志量时尤甚）。 |
| **重置** | 恢复默认：时间默认为「今天 00:00 ~ 当前时刻截断到分钟」或「最近 1 小时」（由实现与后端约定一种默认窗口并在界面脚注说明）；清空 level/event/requestId/keyword；**不自动查询**或 **重置后立即应用一次**（二选一做一致即可，推荐 **重置后自动应用一次** 以减少困惑）。 |

条件变更后结果与内存状态一致；URL 查询串同步为可选（LOG-03）。

**追溯**：D-03、LOG-03。

## 4. 时区与 API 约定（固定决策）

| 层级 | 约定 |
|------|------|
| **界面展示** | 日期时间选择器使用 **浏览器本地时区** 展示与解析用户输入。 |
| **列表列 `ts`** | 格式化为 **本地时区** 的易读字符串（如 `YYYY-MM-DD HH:mm`），tooltip 可显示完整 ISO。 |
| **请求参数** | 客户端将起止时刻编码为 **ISO 8601 字符串**（带偏移或 `Z`，与后端约定一致）发送至 API。 |
| **比较语义** | 服务端将行内 **`ts` 解析为 UTC 瞬时点**（或等价绝对时间），与区间边界比较；**区间开闭**建议 **左闭右闭 `[start, end]`**，若采用 `[start, end)` 须在 API 文档与界面脚注统一标注。 |

界面短文案需让用户能理解：**picker 本地、存储与比较按绝对时间**。  
**追溯**：LOG-03、D-03；供 backend 在 API spec 中逐字对齐。

## 5. 日志列表表格（D-02 / LOG-01）

### 5.1 列定义（桌面优先）

| 列 | 内容 | 备注 |
|----|------|------|
| **时间** | 本地格式化的 `ts` | 缺失或不可解析时显示 **「—」** 或 **「无效时间」**（实现选一并统一）。 |
| **level** | **徽章（Badge）**：`info` / `warn` / `error` 等配色区分；缺失为 **「—」** 或灰色 **「未标注」** 徽章（与筛选「未标注」一致）。 | |
| **event** | 文本，过长省略 + tooltip 全文 | |
| **requestId** | **截断显示**（如前 8 + … + 后 4）；旁 **复制按钮**（图标按钮，`aria-label="复制 requestId"`）。 | LOG-02 |
| **provider / model**（可选） | 若 JSON 中存在且列宽足够则展示；窄屏或 `max-w` 较小时 **隐藏该列**，详情中仍可看。 | 实现按字段存在性适配 |
| **操作** | 文字按钮「查看」或图标「详情」 | 打开抽屉 |

### 5.2 排序与分页

- 默认按 **`ts` 降序**（新在前）。
- **分页** 或 **加载更多** 或 **虚拟列表** 三选一（故事 LOG-01）；设计推荐 **分页**（页码 + 每页条数）以符合「可预测、可分享页」心智。

### 5.3 行交互

- 整行可点击（除复制、按钮外）打开详情，与「查看」等价；**悬停**行背景轻微 `zinc-100/800` 区分。

**追溯**：LOG-01、LOG-02、D-02。

## 6. 详情抽屉（D-04 / LOG-04）

### 6.1 结构

- **顶栏**：标题「日志详情」+ 关闭按钮；可选展示 `event` 或 `ts` 副标题。
- **内容分区**（用 `Divider` 或卡片小节标题区分）：

| 区块 | 建议字段（以实际落盘为准，可增减） |
|------|-----------------------------------|
| **核心（Core）** | `ts`、`level`、`event`、`requestId`（完整可复制一行） |
| **请求（Request）** | 与 HTTP/路由相关字段（如 method、path、status 等，若存在） |
| **消息（Messages）** | `messages`、响应预览类长字段；**默认折叠**，摘要一行（如条数/首条前 N 字） |
| **原始 JSON（Raw JSON）** | 完整对象 **语法着色可选**；**等宽字体**；默认可 **折叠整块**，展开后支持复制全部 |

### 6.2 长字段

- **默认折叠**；展开后内部可 **最大高度 + 内部滚动**，避免抽屉无限拉长。
- 敏感内容心理暗示：折叠区标题可带小号提示「可能含用户消息内容」。

**追溯**：D-04、LOG-04、PRD 5.1。

## 7. 页面与组件状态（D-05）

| 状态 | 触发 | UI 表现 |
|------|------|---------|
| **默认** | 首次进入且已应用默认时间窗 | 表格有数据或空态 |
| **加载中** | 应用筛选、翻页 | 表格 **骨架屏** 或行内 Spinner；**禁用**「应用」防重复提交 |
| **空数据** | 时间窗内无任何行 | 插图/图标 + 文案：**「所选时间范围内暂无日志」** + 建议调整时间或条件 |
| **无搜索结果** | 有查询条件但结果 0 | **「没有符合条件的日志」**；列出当前有效条件摘要（只读 chips）；引导点「重置」 |
| **错误（目录不可读等）** | 5xx 或业务错误码 | **内联 Alert**（红色/destructive）：**「暂时无法读取日志」**；用户可读原因码；**不**暴露服务器绝对路径与栈（LOG-05）。**本迭代不**单独定义「日志 API 403 / 未配置密钥」全页态；若整站网关返回 403，沿用站点统一处理（若有）。 |

**追溯**：D-05、LOG-01、LOG-05。

---

## 8. 安全提示 UI（D-06 / LOG-05）

- **`/console/logs` 页顶**（及 Console 顶栏区域，与配置页同构）：**琥珀色条**，文案强调：日志可能含聊天与模型相关敏感信息、勿暴露公网、建议内网或网关鉴权；**本迭代无**日志 API 密钥输入或「未启用」专页。
- 列表请求失败且非空结果语义时：使用 **错误** 状态（§7），**不**用「无权限」占位冒充空数据。
- 可选：与配置页互链（若产品将其他开关放在配置中）。

**追溯**：D-06、LOG-05。

## 9. 响应式（D-07）

- **桌面**：完整列集 + 右侧抽屉。
- **窄屏**：隐藏 provider/model 列；抽屉全屏；筛选区表单项 **纵向堆叠**；子导航可横向滚动。

**追溯**：D-07。

## 10. 可访问性

| 项 | 要求 |
|----|------|
| **焦点顺序** | 筛选区自上而下 → 表格（行内 Tab 可跳过或仅焦点到操作列，避免每格 Tab，由实现采用 `tabIndex` 策略）→ 打开抽屉后 **焦点移至抽屉内第一个可聚焦元素**（建议关闭按钮或标题）。 |
| **关闭抽屉** | **`Escape`** 关闭并 **焦点归还** 到触发打开的行或「查看」按钮。 |
| **抽屉** | 使用 **焦点陷阱（focus trap）**；背景 **`aria-modal` + `role="dialog"`**（或框架等价）；标题与描述关联 `aria-labelledby` / `aria-describedby`。 |
| **表格** | `<table>` 语义或 `role="grid"` 与行列头 `scope`；排序状态若可变更须-announce。 |
| **图标按钮** | 复制、关闭等须有 **可见 `aria-label`**。 |
