# PRD：助手（Assistants）— 管理端维护与对话侧选用（version=0.3.1）

## 1. 需求摘要

### 1.1 背景

当前对话以**单一应用级默认行为**为主，用户每次新建会话时缺乏可复用的「人设 + 系统提示 + 快捷提问」组合。产品希望引入**助手**实体：由管理员在后台配置，终端用户在**新建会话时**选择其一（或默认无助手），以统一体验并降低重复输入成本。

### 1.2 目标用户

- **管理员**：在现有管理后台（`/console`）维护助手列表与字段。  
- **终端用户**：在**对话页**使用助手；**仅在创建新会话时**绑定助手，绑定后**不可修改**（避免会话中途切换导致上下文与预期不一致）。

### 1.3 核心场景

1. 管理员在后台**新增 / 编辑 / 删除 / 列表展示**助手。  
2. 每个助手包含：**名称**、**提示词**（系统/角色类说明，具体注入方式由后端与设计定稿）、**可选关联知识库**（**可不选**，0 个或多个）、**图标**（**Emoji 字符串**，由管理员输入或选择器录入）、**快捷语**（**可不配置**，0 条或多条；有条目时供对话页展示为快捷按钮，由设计定稿）。  
3. 用户在对话页**新建会话**时，可选择**使用哪个助手**（含「无助手 / 默认」选项）；**选定并创建会话后，本会话不可再更换助手**。  
4. 已有会话进入时，展示当前会话绑定的助手信息（只读，至少名称 + 图标；细节由设计定稿）。  
5. 发送消息时，服务端按会话绑定的助手合并**提示词**及**知识库**（若本迭代对话链路已支持按会话注入 RAG，则启用；若尚未对接，则助手侧先落**配置与存储**，对话注入行为在实现说明中写明「占位/后续迭代」——**须在本 PRD 范围与后端文档中一次性说清**）。

### 1.4 成功指标

- 管理员可完整维护助手字段且无歧义校验（名称必填、提示词必填、快捷语条数上限等由设计/后端约定）。  
- 用户新建会话时助手选择流程清晰；创建后无「切换助手」入口。  
- 会话与助手的绑定关系**持久化**，刷新、重进页面后一致。

## 2. 功能范围

### 2.1 做（In Scope）

| 域 | 内容 |
|----|------|
| 管理端 | 助手 **CRUD**；字段：**名称**、**提示词**、**可选知识库（多选，可空）**、**图标（emoji）**、**快捷语（可选，0～多条）**；列表与表单校验。 |
| 对话端 | **新建会话**流程中**选择助手**（必选「无助手」或某一助手）；**创建后不可改**。 |
| 数据 | 助手独立存储；**会话**记录创建时所选 `assistantId`（可为空表示未使用助手）。 |
| API | 管理端助手 CRUD + 对话侧创建会话时传入 `assistantId`（可选）；与现有会话列表/创建接口对齐（由后端文档定稿）。 |

### 2.2 不做（Out of Scope，本迭代可列）

- 助手**多租户/权限细分**（与现网 console 一致：部署侧控制暴露面）。  
- 助手**版本历史**、**发布草稿**流程（可后续迭代）。  
- 会话中途**动态切换助手**、**同一助手多配置 A/B**。  
- 图标除 **Emoji** 外的自定义图片上传（本迭代仅 emoji 文本）。  
- 若「按助手注入知识库检索」在工程上依赖尚未完成的对话 RAG 链路：可在本迭代仅完成**配置与存储**，对话中**暂不生效**，但须在 PRD/后端说明中**显式**写出，避免误解。

### 2.3 依赖与假设

- 依赖现有 **知识库** 实体与列表接口（若无知识库则多选为空即可）。  
- 依赖现有 **会话** 模型扩展一列或等价字段保存 `assistantId`。  
- 与现有 **意图路由 / 模型选择** 的关系：助手提示词作为**系统侧补充**，与全局配置优先级由**后端文档**定义（避免双 system 冲突）。

## 3. 业务规则（要点）

0. **知识库与快捷语**：二者**均可没有**——助手可以只含名称 + 提示词（+ 可选图标），**不**关联任何知识库、**不**配置任何快捷语；此时对话行为仅受提示词影响，对话页不展示快捷语区域。  

1. **绑定时机**：仅 **`POST /api/chat/sessions`（或项目中等价的「创建会话」）** 请求体携带 `assistantId`（可选）。会话一旦创建，**不允许**再通过 API 或 UI 修改该会话的助手。  
2. **默认值**：用户未选助手时，`assistantId` 为空，行为与当前无助手一致。  
3. **删除助手**：若会话仍引用已删助手，**只读展示**为「助手已删除」或保留快照名称（由设计/后端二选一并写清）；**禁止**因删除助手而静默改变历史消息语义（至少保留会话级快照字段可选，由后端定稿）。  
4. **快捷语**：条数上限、单条长度上限由设计/后端约定；**仅前端快捷入口**，不强制自动发消息（设计可定为「填入输入框」或「直接发送」，二选一并写入设计说明）。
