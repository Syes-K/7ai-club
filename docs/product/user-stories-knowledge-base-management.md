# 用户故事：知识库管理（version=0.1.0）

## US-0.1.0-1：进入知识库管理

**作为** 管理员  
**我希望** 在控制台侧栏有「知识库」入口  
**以便** 独立于应用配置与提示词管理知识库  

**验收要点**

- 从 console 主导航可进入知识库管理相关路由（路径由设计定稿）。  
- 本迭代**无**终端用户聊天中的知识库检索入口。

---

## US-0.1.0-2：维护知识库（库级）

**作为** 管理员  
**我希望** 新建、重命名、删除知识库  
**以便** 按主题或项目划分资料  

**验收要点**

- 新建后列表可见；名称等字段校验失败时有明确提示。  
- 删除前有确认；级联删除**文本文档及对应向量/分块元数据**须在确认文案或文档中说明（由设计定稿）。

---

## US-0.1.0-3：库内文本文档与索引

**作为** 管理员  
**我希望** 在指定知识库内**录入、编辑、列表展示并删除**纯文本文档（**不支持文件上传**）  
**以便** 维护可被系统索引的知识内容  

**验收要点**

- 保存、更新、删除文档后，服务端**分块、Embedding、向量库**与源数据最终一致（允许异步；策略见后端文档）。  
- 本迭代**不要求**对话中检索；**不要求**文件上传。

---

## US-0.1.0-4：服务侧独立模块

**作为** 开发者  
**我希望** 分块、Embedding、向量读写与知识库 CRUD 的核心逻辑集中在**独立服务端模块**内  
**以便** 与 `/api/chat` 解耦、便于测试与后续对接对话 RAG  

**验收要点**

- 代码结构符合 PRD 约定（如独立 `lib/knowledge` 或等价）；聊天路由不包含上述核心业务实现。

---

## US-0.1.0-5：范围边界可识别

**作为** 用户或管理员  
**我希望** 文档与界面说明本版本**已有向量化**但**尚未**在对话中使用  
**以免** 误以为聊天已自动关联知识库  

**验收要点**

- 页脚或说明条：已向量化供后续使用；**对话检索与文件上传**在后续版本（措辞由设计定稿）。
