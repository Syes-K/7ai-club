# 用户故事：对话页模型位置与操作按钮（version=0.0.1）

本文档为 **0.0.1** 增量故事；与 `user-stories-ai-agent.md` 并存，冲突时以 **本迭代 PRD** 为准补充 UI 布局与按钮表现。

---

## US-L1：模型选择处于更合理的布局位置

作为 **使用浏览器的用户**  
我想要 **模型选择不要挤在输入框正上方单独占一行**  
以便 **输入区更聚焦、视觉层次更清晰，同时仍能方便切换模型**。

**验收标准：**

- [ ] AC-L1.1 模型选择控件**不再**固定为「多行输入框紧挨着的上方单独一行」的默认结构（允许与顶栏、底栏工具区或输入同行工具区等方案之一，以设计定稿为准）。
- [ ] AC-L1.2 仍满足 US-1 AC1.1～AC1.4（分组选项、切换后新消息用新路由、历史不丢、默认 `glm-4-flash` 等）——见 `user-stories-ai-agent.md`。
- [ ] AC-L1.3 在 **viewport 宽度较窄**（如移动端常见宽度）时，模型入口仍可见或可触达，不出现不可用的遮挡（允许换行或收折为菜单，由设计规定）。
- [ ] AC-L1.4 请求进行中（`busy`）时，模型控件保持与现网一致的**禁用**策略，避免切换导致状态错乱。

---

## US-L2：主要操作按钮带图标且可用

作为 **使用浏览器的用户**  
我想要 **发送、清空对话、重试等按钮带有清晰图标（可与文字并存）**  
以便 **更快识别操作含义，并与页面其余图标风格一致**。

**验收标准：**

- [ ] AC-L2.1 **发送**按钮：具备图标；禁用态（空输入或 busy）下图标与按钮状态一致。
- [ ] AC-L2.2 **清空对话**按钮：具备图标；无消息或 busy 时禁用与现网逻辑一致。
- [ ] AC-L2.3 **重试**按钮（错误气泡内）：具备图标；`busy` 时禁用与现网一致。
- [ ] AC-L2.4 若某按钮仅展示图标，须提供 **`aria-label`**（或等价可访问名称）；若图标+文字并存，文字可视为可访问名称。
- [ ] AC-L2.5 浅色与深色主题下图标与对比度可接受，焦点环可见（键盘 Tab 可操作发送、清空、重试）。
