# PRD：独立知识库 — 管理、分块与向量化（version=0.1.0）

## 1. 需求摘要

### 1.1 背景

当前系统**尚未**在对话中提供基于知识库的检索增强（RAG）。产品希望建设**独立于对话链路**的**知识库**能力：管理员维护文本资料，服务端对资料执行**分块、Embedding 写入向量库**，为后续「对话中检索与上下文注入」打基础。

本迭代要求知识库相关能力在**服务侧自成模块**（独立目录与边界，**不**将核心逻辑散落在 `/api/chat` 或聊天组件中），便于演进与测试。

### 1.2 目标用户

部署本应用的**管理员/开发者**（与现有 `/console` 场景一致）；不要求多租户。

### 1.3 核心场景（本迭代）

1. 在管理后台查看**知识库列表**（名称、简要信息、**条目数量**、可选**索引/向量化状态**等由设计定稿）。  
2. **新建 / 重命名 / 删除**知识库（删除须二次确认，并明确是否**级联删除库内文本条目及对应向量**）。  
3. 在某一知识库内通过**表单录入纯文本条目**：**新增、编辑、列表展示、删除**；**本迭代不支持文件上传**（含拖拽、附件、批量导入文件等），后续迭代再扩展。  
4. **索引流水线**（在**知识库服务模块**内实现）：对条目的正文（及设计约定的字段）进行**文本分块** → 调用 **Embedding** 模型生成向量 → 写入**向量库**（含更新、删除时的向量同步或重建策略，由后端文档定稿）。  
5. 对话页与 `/api/chat`**不**调用知识库检索；本迭代**不提供**「按知识库回答问题」的入口或 API。管理员可通过**调试/验收手段**（如仅管理端可见的「检索试查」或集成测试，由设计可选）验证向量是否可用——**非**终端用户聊天功能。

### 1.4 成功指标

- 管理员完成文本条目的增删改后，在约定时间内**向量库与块元数据**与源数据一致（允许短暂异步；失败须可感知，见设计）。  
- **Embedding / 分块 / 向量存储**的实现与配置集中在**独立服务端模块**内，对话路由无耦合。  
- 单机部署下**可持久化**；文档标明：**0.1.0 含分块与向量化，不含对话侧 RAG 与文件上传**。

## 2. 功能范围

### 2.1 做（In Scope）

- **服务模块边界**：知识库领域逻辑（库与条目 CRUD、分块、Embedding 调用、向量读写）位于**单独模块**（例如 `src/lib/knowledge/` 或项目约定的等价路径）；**禁止**在 `api/chat` 路由内实现上述核心业务逻辑（允许后续迭代通过**显式 API** 调用该模块）。  
- **管理入口**：与现有控制台并列（例如侧栏「知识库」），路由由设计定稿。  
- **知识库实体**：至少 **id**、**展示名称**、**创建/更新时间**；可选描述、排序等。  
- **文本条目**：表单维护；列表、新建、编辑、删除；校验规则由设计/后端定稿。  
- **文本分块**：策略（固定长度、重叠、分隔符优先级等）与块元数据（条目 id、块序号、原文偏移等）由**后端/设计文档**定稿；需可复现、可删除对齐。  
- **Embedding**：调用约定模型（API Key、base URL、模型名等可通过**环境变量**或**控制台配置**扩展，由后端定稿）；失败重试与降级策略由后端定稿。  
- **向量库**：选型与表结构由后端定稿（须与单机、`data/` 或 SQLite 体系兼容者优先）；支持按库/条目删除或失效对应向量。  
- **服务端 API**：管理端 CRUD + 内部索引更新触发（同步或异步任务由后端定稿）。  
- **鉴权**：与现有 console 一致（默认无登录，文档提示勿暴露公网）。

### 2.2 不做（Out of Scope，本迭代）

- **文件上传**、附件、按文件管理条目。  
- **对话侧集成**：聊天中勾选知识库、向模型注入检索结果、工具调用、流式引用来源等。  
- **混合检索 / 重排序 / 多路召回**等高级 RAG 编排（可后续迭代）。  
- **全文检索 UI**（管理端复杂搜索）、**OCR**。  
- **多用户权限**、**审计日志**（可列后续迭代）。

### 2.3 依赖与假设

- 依赖 Next.js、`/console` ProLayout；可与现有 **SQLite** 并存新表/扩展；向量方案若依赖扩展或独立文件，须在实现说明中写明部署前提。  
- Embedding 依赖**外部 API 或本地模型**时，须在文档中说明配置方式与无密钥时的行为（如禁止写入向量并提示管理员）。

## 3. 待设计项（Design Handoff）

| 序号 | 类型 | 名称 | 说明 |
|------|------|------|------|
| D1 | 信息架构 | 知识库列表与详情 | 列表页 vs 详情页；条目表与编辑表单 |
| D2 | 交互 | 删除与索引 | 删库/删条确认；索引失败、进行中的展示（若有） |
| D3 | 视觉 | 与 console 一致 | 沿用现有管理风 |
| D4 | 空状态与错误 | 无库、无条目、索引失败 | 引导与错误提示 |
| D5 | 可选 | 试检索（仅管理） | 是否提供不经过聊天的检索验证 UI |
