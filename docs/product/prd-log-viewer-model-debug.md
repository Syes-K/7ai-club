# PRD：日志查看器「消息列表 → 模型调试」

**版本**：0.0.7

## 背景

运维在控制台查看聊天落盘日志时，常需根据某条日志里的 `messages` 字段让模型做归纳、排错或对比，但不希望这次调试调用再次写入聊天日志（避免污染与循环）。

## 目标

1. 在日志详情抽屉中，当存在 `messages` 字段时，提供「发送到模型」的调试能力。
2. 请求体由**前端组装**完整 `messages`（含可选的用户指令消息），服务端只做校验、截断与转发。
3. 新增独立接口 **`POST /api/debug`**；**不修改** `POST /api/chat`。
4. 调试请求**不调用** `logChat`（不写 JSONL 聊天日志）。

## 非目标

- 不持久化调试会话与回复。
- 不为 `/api/debug` 单独做鉴权密钥（与现有 `/api/console/logs` 一致，依赖部署侧对 `/console` 暴露面的控制）。

## 验收

- 有 `messages` 的日志行打开详情后可见调试入口；可选 provider / 智谱 model、可选说明文字；流式展示模型输出。
- 调试完成后磁盘聊天日志中不出现对应 `api.request_received` 等条目（在 `chatLoggingEnabled: true` 下亦如此）。
